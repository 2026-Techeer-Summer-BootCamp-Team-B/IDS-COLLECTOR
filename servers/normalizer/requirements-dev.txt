# 로컬 테스트 실행용. requirements.txt를 그대로 -r로 끌어오지 않는 이유는
# correlation-engine/platform-api와 동일 - 그 파일의 `./shared` 줄은 Docker 빌드
# 컨텍스트(servers/) 전제 상대경로라 여기(servers/normalizer/)에서 그대로 실행하면
# 존재하지 않는 경로를 가리킨다.
#
# requirements.txt를 골라서 줄이지 않고 그대로 미러링하는 이유 (2026-07-22 변경) -
# 예전엔 "테스트에 실제로 필요한 것만" 골라 담았는데, app/main.py나 app/geoip.py에
# 새 top-level import가 추가될 때마다(aiokafka, geoip2/redis, fastapi 순으로 세 번
# 연속 - CI 실측) 이 파일이 매번 한 박자 늦게 빠뜨려서 테스트 수집 단계부터
# ModuleNotFoundError로 CI가 깨졌다. correlation-engine/platform-api는 처음부터
# requirements.txt를 통째로 미러링해서 이 문제가 한 번도 발생한 적이 없어서, 같은
# 방식으로 통일한다 - 새 런타임 의존성이 추가돼도 이 파일이 따로 안 빠뜨린다.
#
